# 📦 包装设计稿与实物打样核对系统

上传「设计稿」与「实物打样照片」，系统自动完成图像对齐并把对不上的地方直接高亮框出，帮助在量产前发现印刷不一致（漏印、偏色、元素缺失、移位等）。

## v2.0：纯图像差异为主流程

设计稿常含艺术字/外文字符，OCR 识别易出错。v2.0 起**默认舍弃文字识别，只做图像对比**：

1. **全局对齐**（ORB 特征 + 透视变换）把包装整体摆正；
2. **局部弹性对齐**（分块光流式配准）压平曲面翘曲导致的局部错位，大幅减少描边假高亮；
3. **容配准误差的感知色差**（LAB ΔE，邻域取最小）+ **边缘容差**，只保留成片的真实差异；
4. 连通域聚合成**差异块**，按面积×强度排序，在实物照片上框出。

文字也走像素比对，因此艺术字/外文/多语言小字都能一视同仁地查差异，不受 OCR 误识别影响。
**OCR 文字核对降级为可选项**（侧栏勾选开启），仅在需要逐字核对且文字规整时使用。

### 检测下限与「高灵敏模式」

默认的纯色差通道，判据是"同位置设计稿与实物的颜色差(ΔE)"。若某处花纹与底色颜色很接近，
花纹在与不在的色差都很小，会漏检——实测纯色差的检测下限约 **ΔE 25**（实心块），细线纹理更难。

侧栏可开启 **高灵敏模式（结构/纹理通道）**：在色差之外叠加"局部纹理能量差异"，
把"颜色接近底色、只靠有无纹理区分"的淡纹也纳入检测，两通道**各自独立检测后取并集**
（避免互相抬高阈值）。实测可把检测下限降到约 **ΔE 10**（实心块与细线纹理均可）。
代价：在花纹密集的水彩类设计上，会对"两图都有、但实物印得略软"的细线产生更多误报，
故默认关闭，仅在怀疑有淡纹差异时开启并人工复核。

> 物理下限：检测能力始终正比于花纹与底色的对比度。若花纹与底色**完全同色同质**（零对比），
> 任何被动成像都无信号可用，这不是算法能突破的。

## 在线体验 / 部署

本项目可直接部署到 **Streamlit Community Cloud**：

1. 将本仓库推送到 GitHub。
2. 打开 https://share.streamlit.io ，用 GitHub 登录。
3. 选择本仓库，主文件填 `app.py`，点击 Deploy。

> 首次启动会下载 OCR 模型（约 100MB），需要等待几分钟。

## 本地运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

浏览器打开 http://localhost:8501 即可使用。

## 部署需要的文件

```
app.py                    # 主程序
src/                      # 核心算法模块
  ├── __init__.py
  ├── image_align.py      # 全局图像对齐（ORB + 透视）
  ├── local_align.py      # 局部弹性对齐（分块配准，压制曲面错位）
  ├── pattern_diff.py     # 纯图像差异检测（主流程）
  ├── text_check.py       # 文字核对（OCR，可选）
  ├── pattern_check.py    # 旧版 SSIM 图案检测（保留兼容）
  └── visualization.py    # 报告与可视化
requirements.txt          # Python 依赖
packages.txt              # 系统依赖（apt）
.streamlit/config.toml    # 应用配置（可选）
README.md                 # 说明
```

## 使用要点

- 实物照片尽量**正面垂直平拍**、包装铺平充满画面、光照均匀、避免翘边与反光。
- 侧栏可调节 SSIM / 像素差异 / 文字匹配 三个阈值来控制灵敏度。
- 界面会给出**对齐质量提示**；对齐质量低时结果仅供参考，请人工复核。

## 技术栈

Streamlit · OpenCV · scikit-image（SSIM）· EasyOCR
